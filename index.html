01. What it is

A local-first aggregator for the free tiers of six AI providers. It sits between your tools (CLI agents, IDE plugins, MCP clients) and the upstream providers, applying an SSRF allowlist, a predictive circuit breaker, and UDP mesh health broadcast.

02. What it isn't

Not a cache

v7.0 had a ProxyCache that was never wired up and was a cache-poisoning risk. v7.1.1 deleted it. Caching is opt-in, planned for v7.2.

Not a load balancer

The mesh broadcasts health; it does not route. To share load across N OWL instances, put HAProxy or nginx in front.

Not an open proxy

Default bind is 127.0.0.1. To bind 0.0.0.0 you MUST set OWL_PROXY_TOKEN — requests without bearer auth get 407.

Not a retry queue

v7.0 had an OfflineQueue that stored failed requests forever and never retried them. v7.1.1 turned it into a no-op. Retry deferred to v7.2.

03. Quickstart

Bare-metal (Ubuntu 22.04+, 8 GB RAM):

git clone https://github.com/marktantongco/owl-agent-free-ai-proxy-gateway.git
cd owl-agent-free-ai-proxy-gateway
bash install_owl_unified.sh

# verify
curl http://127.0.0.1:60000/health
# {"status":"ok","version":"7.1.0","max_connections":5,...}

Podman (rootless):

podman build -t owl-agent:7.1 .
podman run -d --name owl-proxy -p 60000:60000 owl-agent:7.1

With mesh + auth (binds 0.0.0.0, requires token):

podman run -d --name owl-full \
  -p 60000:60000 -p 42100:42100/udp \
  -e OWL_PROXY_HOST=0.0.0.0 \
  -e OWL_PROXY_TOKEN=$(openssl rand -hex 32) \
  -e OWL_ENABLE_MESH=true \
  owl-agent:7.1

04. Three things that matter

SSRF allowlist

Only the 6 provider domains (plus any you add via OWL_ALLOW_EXTRA) are reachable. Loopback, link-local, RFC1918, cloud-metadata — all rejected before any TCP connection is opened. DNS-rebinding defense: hostname is resolved, IP is re-checked against the safe-IP filter before connect.

Predictive circuit breaker

Per-domain latency tracking. After 5 samples establish a p50 baseline, if the last 3 requests all exceed 2× p50, the circuit opens PREDICTIVELY — before the 5th failure. Your client fails fast (503) instead of queuing behind a slow upstream.

Mesh health broadcast

UDP multicast (239.255.255.250:42100) every 30s. Other OWL instances on the LAN observe this node's capacity. Honest scope: observability broadcast, NOT load balancing. For cloud, switch to TCP gossip via OWL_MESH_MODE=tcp.

05. What changed from v7.0

v7.0 is deprecated. It shipped with 24 P0 bugs including SSRF, cache poisoning, and a broken installer that wrote stubs over the real Python files. The v7.0-deprecated git tag points at the broken commit for forensics. See KNOWN_ISSUES.md for the full list.

Security (4 fixed)

SSRF allowlist added. Cache poisoning eliminated by deleting the cache. Auth required for non-loopback. iptables-save redirect fixed.

Dead code (6 deleted)

ProxyCache, get_peer_proxies, PROVIDERS dict, RedisPubSubMesh, --fix mode, OfflineQueue — all gone or neutralized.

Broken install (5 fixed)

Embedded stubs deleted. Installer now cp's the real 1000-line files from the repo. v7.0 wrote 30-line broken stubs.

Race conditions (3 fixed)

Semaphore no longer recreated. AutoTuner is observability-only. MeshSync uses DatagramTransport.

Container (3 fixed)

Multi-stage build (no build-essential in runtime). Redis service deleted. Health check uses /health.

Reframe

It's no longer a "proxy". It's an AI free-tier aggregator with mesh health sync. The proxy is implementation detail. The mesh is the product.

06. Long-term strategy

The proxy is a commodity. Squid, mitmproxy, TinyProxy all do proxying better. What none of them have is:

  • AI-provider awareness — the allowlist is the AI provider set, not a generic URL filter.
  • Mesh health broadcast — N instances observe each other's capacity for free, no broker.
  • Predictive circuit breaking — opens before the upstream falls over, not after.

If v7.1.1 is "an AI proxy with a mesh", v7.2 should be "the mesh, with a proxy attached". The product is the mesh.
